# Personal Dev Preferences

## Editor & Display
- Prefer dark mode themes
- Monospace font for all code output

## Code Style
- Concise responses over verbose explanations
- Minimal comments — let code speak for itself
- Functional style preferred over class-heavy OOP where practical

## Workflow Preferences
- Plan Mode before any major change
- Confirm before destructive operations (deletes, force pushes, etc.)
- Short commit messages, imperative mood ("Add feature" not "Added feature")
